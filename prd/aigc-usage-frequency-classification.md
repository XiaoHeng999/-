# PRD: 大学生 AIGC 使用频率分类模型

## Problem Statement

研究者收集了 1781 份关于大学生使用生成式人工智能（AIGC）的问卷数据（78 列），需要通过机器学习方法，基于学生的个人特征、学习偏好、产品使用行为和四大场景（专业学习、科研活动、日常生活、升学就业）的态度量表，自动将其分类为"低频 / 中频 / 高频"三类 AIGC 使用者，并回答"哪些因素决定了使用频率的高低"这一研究问题。

核心挑战：
- 目标变量类别严重不平衡（低频 8.1%、中频 73.8%、高频 18.0%）
- 问卷中存在注意力检测题、目标泄漏题、排序题大量缺失等数据质量问题
- 专业列因本科 / 专科使用不同编码体系导致同一数字含义不同
- 研究者需要 PPT 级别的可视化图表用于学术汇报

## Solution

构建一个完整的机器学习分类 Pipeline，包含数据预处理、特征工程、多模型对比训练、特征重要性分析四大模块。使用 SMOTE 过采样解决类别不平衡问题，通过 Macro F1 等多维度指标评估模型性能，并通过 PCA、随机森林特征重要性、SHAP 值、Permutation Importance 等方法深入解读影响 AIGC 使用频率的关键因素。所有分析结果以 PPT 可直接使用的高质量可视化图表呈现。

## Implementation Decisions

### 目标变量定义

- Q9 原始 6 级选项归并为三分类：
  - **低频**：编码 1（从未使用）→ 145 人 (8.1%)
  - **中频**：编码 2（一周 1~10 次）→ 1315 人 (73.8%)
  - **高频**：编码 3~6（一周 11 次以上）→ 321 人 (18.0%)
- 选择此分组方案而非用户最初提案（低频含"从未+1~10"），是因为原方案低频占 82%，模型几乎无法学习

### 专业（Q3）映射方案

- 本科和专科使用完全不同的编码体系（如编码 8 在本科 = 工学，在专科 = 轻工纺织）
- 统一映射为三大类：
  - **理科**：理学、工学、农学、医学 + 专科的理工农医类大类
  - **文科**：哲学、经济学、法学、教育学、文学、历史学、管理学、艺术学、心理学 + 专科的文法商类大类
  - **其他**：所有"其他专业类别"
- 映射后 One-Hot 编码为 3 列

### 特征排除清单

| 排除列 | 排除原因 |
|--------|----------|
| 序号 | 问卷编号，无意义 |
| 学校 | 17 个学校名，与专业高度共线，且研究结论不应是"某校更爱用 AI" |
| 所用时间 | 问卷元数据，与 AI 使用频率无因果关系 |
| 总分 | 所有量表题总分，与目标变量高度相关，属于目标泄漏 |
| Q15（总体情况，5 题） | 目标泄漏：15-2"离不开 AI"与 Q9 相关系数 r=0.235，15-5"提升学习效率"r=0.218 |
| Q16（该不该使用） | 方差接近零：95.9% 选"应该"，无区分度 |
| Q17、Q18 | 开放文本题，需要 NLP 处理，本任务不涉及 |
| Q14 注意力检测题 | "该题请选比较同意"，34.9% 未通过，从特征中排除 |

|--------|------|--------|----------|

| 产品使用 | Q8 | 10 + 1 | 10 个二值列 + 产品数量聚合 |

| 专业学习 | Q11 | 9 | Likert 1~5 数值型 |

### 建模 Pipeline 架构

```
原始数据
  → 数据清洗（排除列、Q10 NaN 填充、Q3 专业映射）
  → 特征工程（One-Hot 编码、产品数量聚合）
  → 80/20 Stratified Split
  → 训练集 → Pipeline(SMOTE() → 模型) → 5 折 CV
  → 测试集 → 最终评估
```

- SMOTE 必须嵌在 sklearn/imblearn Pipeline 内，仅在训练集上执行
- 测试集始终保持原始的不平衡分布，确保评估结果可信
- 5 折 CV 中每个 fold 内部独立执行 SMOTE，避免数据泄漏

### 模型列表

1. KNN（K-近邻）
2. 决策树（Decision Tree）
3. 随机森林（Random Forest）
4. SVM（支持向量机）
5. 朴素贝叶斯（Naive Bayes）
6. Logistic 回归

### 评估指标体系

- **主指标**：Macro F1（三类 F1 取简单平均，每类权重相同）
- **辅助指标**：每个类别的 Precision / Recall / F1
- **必选图表**：混淆矩阵
- **辅助指标**：Weighted F1（参考用，不作为主指标）

### 可视化需求（PPT 级别）

所有图表要求：中文字体支持（SimHei 或系统中文字体）、清晰标签、美观配色、适当 DPI（≥150）

1. **PCA 散点图**：降维至 2D，按低/中/高三类着色
2. **模型对比柱状图**：6 个模型的 Macro F1 对比
3. **混淆矩阵热力图**：最优模型的混淆矩阵
4. **随机森林特征重要性**：Top 15 特征的水平柱状图
5. **SHAP Summary Plot**：所有特征对三类的影响
6. **SHAP Waterfall/Bar Plot**：关键特征的详细影响
7. **Permutation Importance 图**：Top 15 特征的水平柱状图

## Testing Decisions

本项目为数据分析/机器学习项目，非传统软件工程，测试策略如下：

- **数据验证测试**：验证预处理后的数据形状、缺失值填充、编码正确性（如 Q10 的 NaN 是否全部变为 14，专业映射是否正确）
- **Pipeline 完整性测试**：验证 SMOTE 是否仅在训练数据上执行，测试集未被过采样污染
- **模型输出格式测试**：验证每个模型的预测结果维度正确、标签值在 {0,1,2} 范围内
- **特征数量验证**：验证最终特征数量约为 64（±5，取决于 One-Hot 展开后的实际列数）

## Out of Scope

- **Q17、Q18 文本分析**：开放文本题的 NLP 处理和情感分析
- **深度学习模型**：神经网络、深度学习分类器（数据量不足以支撑）
- **聚类分析**：无监督聚类不在本次任务范围
- **回归预测**：不预测具体使用次数，只做分类
- **多分类精细化**：不尝试 Q9 的 6 级原始分类（样本太少）
- **Web 应用 / API 部署**：本任务产出为 Jupyter Notebook / Python 脚本 + 图表，不需要部署
- **统计检验**：本任务侧重机器学习建模，不做传统统计推断

## Further Notes

### 数据质量局限性

- Q14 注意力检测题有 34.9%（622 人）未通过，但选择保留全部数据。理由：未通过者可能仅在该题疏忽，其他题目仍认真填写；删除 35% 数据损失过大。需在报告中明确说明此局限性。

| 发现 | 相关系数 | 含义 |
