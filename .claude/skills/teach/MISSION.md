# Mission: Loop Engineering 概念理解

## Why
用户希望深入理解 Loop Engineering 这一 2026 年兴起的新范式，以便与同行进行有质量的讨论，并评估其在软件工程中的位置。不是立即在实践中使用，而是建立清晰的概念模型。

## Success looks like
- 能用一句话准确解释 Loop Engineering 与 Prompt Engineering 的区别
- 能说清楚"循环"和"自动化"的关键差异
- 能列举至少 3 种必须的"护栏"及其存在原因
- 能判断一个场景是否适合用 Loop Engineering

## Constraints
- 中文为主要语言
- 偏好概念理解而非动手实践
- 从零开始，没有 Agent 系统的前置知识假设

## Out of scope
- 具体工具的使用教程（Claude Code / Codex CLI 的操作细节）
- 多 Agent 编排的架构设计
- Token 成本优化的具体策略
