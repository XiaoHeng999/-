Loop Engineering 术语表

2026 年 AI Agent 工作范式的核心概念速查

核心概念

Loop Engineering 循环工程
设计和维护让 AI Agent 自动触发、执行、验证、重试的循环系统的实践。不再是人去 prompt Agent，而是让系统去 prompt Agent。
避免与：自动化、脚本调度混淆
Agentic Loop 智能体循环
一个由触发器（Trigger）和可验证目标（Verifiable Goal）组成的循环。Agent 持续运行直到目标达成或停止条件触发。
Trigger 触发器
启动循环的事件。三种类型：事件触发（PR 创建、CI 失败）、定时触发（cron）、人工触发（手动输入命令）。
Verifiable Goal 可验证目标
循环的停止条件。必须是可被自动验证的，如"所有测试通过"、"CI 绿灯"。模糊目标（如"让代码更好"）不适合循环。

循环内部机制

5 阶段循环
感知 → 推理 → 规划 → 执行 → 观察。每个循环迭代经过这五个阶段，然后回到感知，直到目标达成。
ReAct
Reasoning + Acting 的缩写。每一步同时产生推理痕迹和具体行动。2022 年由 Princeton 和 Google Research 提出，是大多数 Agent 循环的基础模式。
Ralph Loop
一种简单但有效的循环模式：Agent 在无限 shell 循环中运行，每次迭代重置上下文窗口，状态保存在文件系统中。解决上下文溢出和过早退出问题。

护栏（Guardrails）

Guardrails 护栏
防止 Agent 循环失控的硬性约束机制。没有护栏的循环等于"一台非常自信的 token 焚化炉"。
迭代上限 Iteration Cap
最大循环次数。超过后 Agent 必须停止并报告当前状态，无论是否达成目标。
Token/成本预算
每次运行的硬性花费上限。防止 Agent 无限重试导致巨额账单。
无进展检测 No-Progress Detection
如果连续几轮迭代的输出没有变化，自动退出。防止"忙碌但无效"的循环。
人工检查点 Human-in-Loop Checkpoint
在不可逆操作（数据库写入、部署、外部 API 调用）前必须由人工确认。
终止条件 Termination Criteria
在循环启动前就定义好的"完成"标准。使用可自动验证的检查（运行测试、类型检查），而非 Agent 自我评估。

演进谱系

Prompt Engineering 提示工程
优化单次人机交互的技巧。2023 年主流范式。Loop Engineering 的前身。
Context Engineering 上下文工程
管理 Agent 能看到什么信息、使用什么工具的技术。2024 年的概念。Loop 的关键组件。
Harness 脚手架
编排逻辑、内存管理、成本控制、可观测性的基础设施层。是 Loop Engineering 的运行时环境。

常见失败模式

Infinite Loop 无限循环
没有目标验证机制，Agent 不断"优化"永远不停。
Goal Drift 目标漂移
Agent 开始追求与原始目标相关但不同的目标。由模糊规格或工具结果误导引起。
Silent Failure 静默失败
Agent 产生看起来正常的输出，但实际上没有任何进展。最难检测的失败模式。
Token Cost Explosion Token 成本爆炸
单 Agent 消耗约 4 倍标准 chat 的 token，多 Agent 系统约 15 倍。无约束的循环可导致巨额账单。
