Loop Engineering 是什么？
从 Prompt 到 Loop 的范式转变
"我不再 prompt Claude 了。我有循环在运行，它们来提示 Claude，来决定要做什么。我的工作是写循环。" — Boris Cherny，Claude Code 负责人，Anthropic

一句话定义

核心定义

Loop Engineering 是设计和维护让 AI Agent 自动运行、自我检查、自我重试的循环系统的实践。你不再是"提示 Agent 的人"，而是"设计让 Agent 自动运作的系统的人"。

这个概念在 2026 年 6 月爆发，原因是 AI 编码工具的两位核心人物——Claude Code 的 Boris Cherny 和 OpenClaw 的 Peter Steinberger——几乎同时提出了同一个观点：你不应该再手动 prompt 你的编码 Agent 了。

一个直觉性的比喻

想象你是一位工厂经理：

Prompt Engineering（2023 年的模式）就像你站在流水线旁边，每完成一个步骤就亲手告诉工人"下一步做这个"。你必须在场，你必须不停地说话。

Loop Engineering（2026 年的模式）就像你设计了一条自动流水线——你设定好了原料入口（触发器）、质量检测门（可验证目标）、异常处理规则（护栏），然后你就可以回家了。流水线自己运行，自己检查，自己调整。

关键区别：自动化流水线执行固定步骤；Loop 内部有一个会思考和决策的 Agent，它能判断"目标还没达成，我需要换个方法试试"。

循环 vs 自动化：为什么不只是个高级 Cron Job？

这是最容易混淆的地方。一个 cron job 每小时运行一次——这不就是循环吗？

自动化 (Automation) 循环 (Loop)
做什么 执行固定脚本，步骤不变 Agent 评估当前状态，选择下一步行动
遇到错误 失败就停了 Agent 分析错误原因，调整策略重试
判断完成 脚本跑完就"完成" Agent 主动检查目标是否达成
决策者 写脚本的人 循环内的Agent

参考：Loop Engineering Explained — Louis-François Bouchard

一个循环的内部结构

┌──────────────────────────────────────────────────┐ │ 触发器: 每天早上 9 点 / PR 创建 / 手动启动 │ │ ↓ │ │ 可验证目标: 所有测试通过，CI 绿灯，代码已提交 │ │ ↓ │ │ ┌─── 循环开始 ───────────────────────────────┐ │ │ │ │ │ │ │ 1. 感知 ← 读入当前状态、错误信息、工具结果 │ │ │ │ 2. 推理 ← 分析：目标达成了吗？还差什么？ │ │ │ │ 3. 规划 ← 决定下一步做什么 │ │ │ │ 4. 执行 ← 写代码、跑测试、调工具 │ │ │ │ 5. 观察 ← 接收结果，更新理解 │ │ │ │ ↓ │ │ │ │ 目标达成了？ │ │ │ │ ├── 是 → 开 PR，更新 ticket，循环结束 ✓ │ │ │ │ └── 否 → 回到第 1 步继续 │ │ │ │ │ │ │ └─────────────────────────────────────────────┘ │ │ ↓ │ │ 护栏: 迭代上限 / Token 预算 / 无进展检测 │ └──────────────────────────────────────────────────┘

你在设计这个系统的时候做了一次性的工作：定义目标、选择触发器、设置护栏。然后它自己跑。你没有手动 prompt 七次——你设计了一个七步系统，一次就搞定。

护栏：为什么 Loop 没有护栏就是灾难

这是一个关键点。Bouchard 的原文说得很到位：

"如果目标验证不存在，那你建造的不是循环——是一台非常自信的 token 焚化炉。" — Loop Engineering Explained

护栏（Guardrails）是防止循环失控的硬性约束。不是可选的——每个生产级循环都必须有。

为什么？因为 LLM 没有"我完成了"的内在概念。如果你不给它明确的停止条件，它会永远跑下去——不断"优化"、不断重试——直到你的钱烧光。

🔢
迭代上限
最大循环次数。超过后必须停止并报告当前状态。
💰
Token / 成本预算
每次运行的硬性花费上限。已知案例：Agent 调用一个坏工具，5 分钟内调了 400 次。
📉
无进展检测
连续几轮输出没变化就退出。防止"忙碌但无效"的循环。
👤
人工检查点
不可逆操作（部署、数据库写入）前必须人工确认。
✅
终止条件
启动前就定义好"完成"的标准。用自动测试验证，不用 Agent 自我评估。
⏰
超时
任务级和工具调用级都要有超时。防止卡死。

🧠 检查你的理解

问题 1：Loop Engineering 和 Prompt Engineering 最本质的区别是什么？

  • Loop 用的模型更大，Prompt 用的小模型
  • Loop 是多轮对话，Prompt 是单轮对话
  • Prompt 优化单次交互，Loop 设计可自动重复运行的系统
  • Loop 只适用于编码，Prompt 适用于任何任务

问题 2：以下哪个是"可验证目标"的好例子？

  • 所有单元测试通过，CI 构建成功
  • 代码质量显著提升
  • Agent 认为任务已经完成
  • 用户满意度和代码可读性改善

问题 3：为什么每个循环都需要"护栏"？

  • 因为法律要求所有 AI 系统都有安全措施
  • 因为 LLM 没有内在的"完成"概念，无约束会永远运行下去
  • 因为护栏可以提高 Agent 的推理能力
  • 因为护栏能减少 Agent 使用的 token 数量
